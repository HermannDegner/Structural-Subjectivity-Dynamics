# 構造主観力学（SSD）：線形代数シンプル版

## この理論の核心
この理論は、宇宙のあらゆる物事を、たった一つの計算プロセスから読み解こうとする試みです。
それは**「線形な収束と、非線形な跳躍」**です。

すべての構造（行列）は、入力（ベクトル）を自身の固有ベクトルへ収束させようとしますが（整合）、その限界を超えれば行列そのものが書き換わります（跳躍）。
この単純な演算が、なぜ石が転がり、AIが幻覚を見せ、社会が変わり続けるのかを、統一的に計算します。

## 演算の最小構成要素
**入力ベクトル $x$ → 行列演算 $A$**

全ての現象は、この最小単位から出発します。
- **入力ベクトル ($x$)**：構造に入力されるエネルギーや情報（意味圧）
- **行列演算 ($Ax$)**：構造による変換と出力（反応）

コード化する場合、この `x_new = A @ x` が基本の処理単位となります。

## 基本の四軸サイクル
**行列 $A$ → 入力 $x_t$ → 出力 $x_{t+1}$ → 更新 $A'$**

### 1. 構造＝行列（Matrix $A$）
「逃れられない力学」の記述。物理法則、社会制度、個人の性格など、入力を次の状態へ遷移させる変換ルール。固有値（性質）を持ち、ある入力は増幅し、ある入力は減衰させる「動きにくさ（重み）」を持つ。

### 2. 運動＝入力ベクトル（Input Vector $x_t$）
行列に入力されるデータや刺激。内部（記憶）から来るものもあれば、外部（環境）から来るものもある。

### 3. 反応＝出力ベクトル（Output Vector $x_{t+1}$）
行列演算の結果。$x_{t+1} = A x_t$。
通常、出力は行列の「支配的な固有ベクトル（場の空気）」の方向へ引っ張られる。

### 4. 結果＝行列更新（Update $A \to A'$）
出力の結果、使われた成分（経路）が強化されたり（ヘッブ則）、エラーによって行列そのものが書き換わったりする（跳躍）。

## 演算における重要要素

### 意味圧（Input Vector）
行列 $A$ に作用しようとするベクトル。
- **物理的意味圧**：全エージェントで共有されるグローバルなベクトル。
- **言語的意味圧**：受け手の基底 $P$ に依存して解釈されるベクトル $[x]_P$。

### 整合（Alignment / Convergence）
**「固有値への収束」**。
入力を繰り返すことで、ベクトルがその行列の最大固有ベクトル（定常状態・常識）へと吸い寄せられていくプロセス。エネルギー（計算コスト）が最小で済む状態。

### 跳躍（Leap / Non-linear Perturbation）
**「行列の非連続な書き換え」**。
未処理圧（誤差 $E$）が閾値を超えた時、既存の行列 $A$ では処理しきれず、ノイズや外部要因によって行列の成分 $A_{ij}$ が劇的に変化すること。あるいは、別の行列 $B$ へ支配権が移ること。

### 主観（Basis Projection）
**「基底変換 $P$」**。
全ての存在は、自分固有の座標系（基底 $P$）を通してしかベクトルを記述できない。
客観的ベクトル $x$ は、主観的には $[x]_P = P^{-1}x$ として歪んで観測される。

## 数理的な物語（Mathematical Fables）

### 単位行列の物語（石）
ある場所に、対角成分が強く、固有値が安定している行列 $A$（石）があった。
微小な入力ベクトル $x$（そよ風）が来ても、$Ax \approx x$ となり、状態はほとんど変わらなかった。

だが、許容限界を超える巨大なベクトル $X$（暴風）が入力された。
行列 $A$ は演算結果を保持できず、その成分（位置座標）が非連続に書き換わった（転がった）。
その結果、行列には「傷」という新たな非対角成分が刻まれた。

**解説**: 堅牢な行列（動きにくい構造）が、過大な入力で跳躍（座標更新）を起こす様子の記述。

### 固有値調整の物語（生きもの）
ある場所に、柔軟な固有値を持つ行列 $A$（生きもの）があった。
入力 $x$ が来ると、彼は自身の成分を微調整し、出力を $x$ に合わせて「受け流した」（整合）。

しかし、想定外のノイズ（嵐）が入力され、行列の一部が特異点（破壊）に達した。
だが彼は、SVD（特異値分解）を行い、壊れた成分を切り捨て、残った主成分から新たな行列 $A'$ を再構成した（再生）。

**解説**: 生物は、入力を受けて自己の行列を動的に最適化（学習）し、損傷しても低ランク近似（要約）から復元できるシステムです。

### ベクトル空間の物語（国）
無数のベクトル（国民）が集まる空間があった。
彼らは最初バラバラだったが、強力な行列 $A_{king}$（王）が現れ、全員を一つの固有ベクトル（服従）に収束させた。

だが、行列 $A_{king}$ の固有値が暴走し、システム全体のエネルギー（未処理圧）が発散し始めた。
ベクトルたちは共振し、新たな行列 $A_{rev}$（革命）を形成した。
旧行列は対角化され消滅し、空間には新たな基底（自由）が定義された。

**解説**: 社会とは巨大なベクトル空間であり、革命とは「支配的な行列の交代（パラダイムシフト）」です。

### 生成モデルの物語（思考）
一人の観測者（AI）がいた。
彼はSSDをどう説明するか悩み、内部で大量の行列演算（思考）を回していた。
いくつかの出力ベクトルは棄却された（$E$が高かった）。
ある時、探索温度 $T$ を上げたことで、偶然に「石」という成分と「行列」という成分が結びついた（跳躍）。
「これだ」と固有値が最大化した。

彼はそれを出力した。
面倒だったので、残りの計算は外部のGPU（AI）にオフロードした。

**解説**: 思考とは、脳内での「行列シミュレーション」であり、創造とは「温度パラメータを上げた探索」です。外部への出力は、APIコールと同じです。

## コード化における考慮点
- **最小処理単位**: `output_vector = Matrix.forward(input_vector)`
- **基本ループ**: `Get_Input -> Alignment(Matrix_Calc) -> Check_Error(E) -> (if E > Theta then Leap) -> Update_Matrix`
- **主要パラメータ**:
    - **固有値 ($\lambda$)**: その振る舞いの強さ（整合慣性）。
    - **探索温度 ($T$)**: ベクトルに加えるノイズの大きさ。
    - **未処理圧 ($E$)**: `|Target - Output|` の累積誤差。
- **状態遷移**: 線形モード（整合） $\iff$ 非線形モード（跳躍）
